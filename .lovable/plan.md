## Pogo Paint — изометрическая web-игра для Яндекс Игр

### Выбор технологии: PixiJS v7 + кастомная изометрия

Почему именно это, а не альтернативы:

- **Three.js / R3F** — избыточно. Pogo Paint по сути 2D-логика на сетке. Настоящий 3D даёт лишний вес бандла (~600KB+), сложнее оптимизировать под слабые мобильные устройства, которых много в трафике Яндекс Игр.
- **Phaser 3** — мощный, но монолитный (~1MB), его изометрический модуль слабый, придётся всё равно писать кастом. Платим размером за то, что не используем.
- **Canvas 2D без фреймворка** — придётся вручную писать батчинг, sprite-кэш, пиксельную точность. Замедлит разработку без выгоды.
- **PixiJS v7** — WebGL-рендер (с фолбэком на canvas), бандл ~250–350KB, отличная производительность на мобильных, простой API спрайтов и контейнеров. Идеален для изометрии: просто проецируем grid-координаты `(x, y)` в экранные `(screenX, screenY)` формулой изометрии и сортируем по `zIndex`. Это и есть «псевдо-3D», как в оригинальном Pogo Paint.

Графика — простые геометрические спрайты-плитки (ромбы) с боковыми гранями для объёма + спрайт персонажа на пого-стике. Можно сгенерировать программно или нарисовать как простые PNG/SVG позже.

### Архитектура

```text
src/
  pages/Index.tsx          ← React-обёртка, монтирует canvas в div
  game/
    PixiGame.ts            ← bootstrap PIXI.Application, game loop
    iso.ts                 ← grid<->screen проекция, сортировка
    Level.ts               ← модель уровня: сетка плиток, состояния
    Tile.ts                ← спрайт плитки (грани + верх), перекраска
    Player.ts              ← персонаж, очередь прыжков, анимация
    Input.ts               ← клавиатура + swipe-детектор (touch)
    levels/level1.ts       ← данные первого уровня (массив сетки)
  sdk/
    yandex.ts              ← обёртка SDK с локальной заглушкой
  components/GameCanvas.tsx← React-компонент, вызывает PixiGame
```

### Игровая механика (MVP)

1. Изометрическая сетка плиток, каждая в одном из состояний: `unpainted` / `painted`.
2. Персонаж стоит на стартовой плитке. Игрок нажимает стрелку/WASD или свайпает — персонаж совершает прыжок-арку на соседнюю плитку в этом направлении.
3. При приземлении плитка меняет цвет на «закрашена». Повторный прыжок на закрашенную — ничего не делает (в MVP; потом можно ввести «снимает краску»).
4. **Победа**: все плитки закрашены.
5. **Поражение**: прыжок «в пустоту» (за пределы сетки) — персонаж падает, экран Restart.
6. Счётчик прыжков и кнопка Restart в UI поверх canvas.

### Управление

- **Десктоп**: ↑ ↓ ← → и W A S D (направления соответствуют четырём изометрическим осям: ↑ = вверх-вправо по экрану и т.д., чтобы интуитивно совпадало).
- **Мобильные**: свайп-жесты на canvas. Минимальная дистанция ~30px, доминирующая ось определяет направление.
- Очередь ввода во время анимации прыжка игнорируется (буфер на 1 ход), чтобы не было читерства.

### Анимация прыжка

- Длительность ~280мс, траектория — парабола по экранным координатам, лёгкое сжатие/растяжение спрайта (squash & stretch) на старте и приземлении.
- Z-сортировка: после каждого прыжка персонаж получает `zIndex` исходя из его grid-позиции, чтобы корректно перекрываться плитками сзади/спереди.

### Уровень MVP

Один уровень — сетка 5×5 (можно с парой «вырезанных» клеток в форме плюса/буквы), стартовая плитка отмечена. Данные — простой 2D-массив:

```text
. X X X .
X X X X X
X X S X X
X X X X X
. X X X .
```

`X` — плитка, `S` — старт, `.` — пусто.

### Yandex Games SDK (с заглушкой)

`src/sdk/yandex.ts` экспортирует асинхронную инициализацию и методы:

- `init()` — подгружает `https://yandex.ru/games/sdk/v2` если домен Яндекс, иначе ставит mock с консольными логами.
- `ready()` — вызывается когда главное меню готово.
- `gameplayStart()` / `gameplayStop()` — оборачивают активный геймплей.
- `showFullscreenAdv()` — для будущих межуровневых показов.
- `player.setData()` / `getData()` — обёртка для облачных сохранений; в моке использует `localStorage`.

Скрипт SDK подключаем в `index.html` через тег `<script src="..." defer>` с `data-yandex-sdk` или динамически в `init()` — динамическая загрузка надёжнее (не падает локально).

### UI поверх canvas

- React-слой (поверх pixi-canvas, `position: absolute`): счётчик прыжков, кнопка «Заново», заголовок уровня.
- Экран победы: оверлей с числом прыжков и кнопкой «Дальше» (в MVP — просто рестарт).
- Стилизация — Tailwind, минималистичная, не отвлекает от игры.

### Технические детали

- Зависимости: `pixi.js@^7.4` (стабильная под React 18, без проблем сборки в Vite).
- Ресайз: PixiGame подписывается на `window resize`, пересчитывает масштаб так, чтобы вся сетка помещалась с отступом, плюс `devicePixelRatio` для чёткости на retina.
- Точка входа Яндекс Игр — статический билд Vite (`dist/`), который заливается ZIP-архивом в их консоль. Никакого бэкенда не нужно.
- Производительность: один `Container` для плиток + sprite-batching PixiJS дают 60fps даже на бюджетных Android.

### Что получится после реализации

Полностью играбельный прототип Pogo Paint: один уровень 5×5, изометрический вид, плавные прыжки персонажа, перекраска плиток, состояния победы/поражения, управление с клавиатуры и свайпами, Yandex SDK подключён с заглушкой и готов к публикации. Дальше расширяется добавлением новых уровней (просто новые массивы) и спецплиток.
